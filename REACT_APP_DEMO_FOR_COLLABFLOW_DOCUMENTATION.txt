1. PROJECT OVERVIEW
================================================================================

PROJECT NAME: CollabFlow
DESCRIPTION: CollabFlow is a sophisticated project management platform that
combines AI-powered task generation with human collaboration and human-in-the-loop
(HITL) workflows. The application follows a "Notion meets Claude" aesthetic,
emphasizing clean design, generous whitespace, and intelligent color usage to
distinguish between different task types and workflow states.

CORE PHILOSOPHY:
- AI-Powered Efficiency: Leverage AI to generate and organize tasks
- Human Collaboration: Enable seamless team collaboration and communication
- Human-in-the-Loop: Maintain human oversight and decision-making in critical processes
- Elegant Design: Provide a sophisticated, intuitive user interface
- Theme Support: Full light and dark mode support with persistent user preferences

TARGET USERS:
- Project managers and team leads
- Product teams using AI-assisted workflows
- Organizations requiring human oversight in automated processes
- Teams seeking a modern alternative to traditional project management tools

2. APPLICATION ARCHITECTURE
================================================================================

TECHNOLOGY STACK:
- Framework: Next.js 15+ (App Router)
- UI Library: React 18+
- Styling: Tailwind CSS v4
- Component Library: shadcn/ui
- Icons: Lucide React
- State Management: React Hooks (useState, useContext)
- Theme Management: Custom ThemeProvider Context
- Font System: Tahoma (headings), Montserrat (body text)

ARCHITECTURAL PATTERNS:
- Server Components: Used for layout and data fetching
- Client Components: Used for interactive features and state management
- Context API: Used for theme management and global state
- Component Composition: Modular, reusable components
- Layout Wrapper Pattern: Consistent layout across all pages

KEY ARCHITECTURAL DECISIONS:
1. LayoutWrapper Component: Provides consistent sidebar, header, and main content
   layout across all pages, ensuring uniform navigation and spacing.

2. ThemeProvider Context: Manages light/dark theme state, persists to localStorage,
   and applies theme to document root for CSS variable switching.

3. CSS Variables: All colors and spacing use CSS variables defined in globals.css,
   enabling seamless theme switching without component re-renders.

4. Modular Components: Each feature (dashboard, projects, tasks, schedule) has
   dedicated component folders with specialized sub-components.

5. THEME SYSTEM & COLOR SPECIFICATIONS
================================================================================

THEME ARCHITECTURE:
The CollabFlow application uses a comprehensive CSS variable-based theme system
that supports both light and dark modes. All colors, spacing, and design tokens
are defined as CSS variables in globals.css, enabling seamless theme switching
without component re-renders.

THEME PROVIDER (components/theme-provider.tsx):
- Manages theme state (light/dark)
- Persists theme preference to localStorage
- Applies theme class to document root
- Provides useTheme hook for components
- Prevents flash of unstyled content (FOUC)

BRAND COLORS (Primary Palette):
These are the core brand colors used throughout the application:

1. BITTERSWEET (#E74C3C)
   - Usage: Urgent items, errors, warnings, danger actions
   - Light Mode: #E74C3C (vibrant red-orange)
   - Dark Mode: #E74C3C (maintains vibrancy)
   - Semantic: --bittersweet
   - Examples: Error badges, delete buttons, urgent notifications

2. EGGPLANT (#4A235A)
   - Usage: Primary brand color, dark accents, premium feel
   - Light Mode: #4A235A (deep purple)
   - Dark Mode: #6B3A7D (lighter purple for visibility)
   - Semantic: --eggplant
   - Examples: Sidebar background, primary buttons, brand elements

3. GLAUCOUS (#5B8DEE)
   - Usage: AI-related items, primary actions, interactive elements
   - Light Mode: #5B8DEE (bright blue)
   - Dark Mode: #7BA3F5 (lighter blue for contrast)
   - Semantic: --glaucous, --color-ai
   - Examples: AI task badges, primary buttons, links, focus states

4. TEA GREEN (#2D5A3D)
   - Usage: Human tasks, completed items, success states
   - Light Mode: #2D5A3D (dark green for contrast)
   - Dark Mode: #7FB069 (lighter green for visibility)
   - Semantic: --tea-green, --color-human, --color-success
   - Examples: Human task badges, completed checkmarks, success messages

5. ORANGE PEEL (#FF9500)
   - Usage: Human-in-the-loop (HITL) items, pending actions, warnings
   - Light Mode: #FF9500 (vibrant orange)
   - Dark Mode: #FFB84D (lighter orange for contrast)
   - Semantic: --orange-peel, --color-hitl
   - Examples: HITL task badges, pending items, attention-needed indicators

NEUTRAL COLORS (Background & Text Palette):
These colors form the foundation of the light and dark themes:

LIGHT MODE NEUTRALS:
- --background-50: #FFFFFF (pure white, main background)
- --background-100: #F8F9FA (very light gray, secondary background)
- --background-200: #F0F2F5 (light gray, tertiary background)
- --background-300: #E8EAED (light gray, borders and dividers)
- --text-900: #1A1A1A (near black, primary text)
- --text-800: #333333 (dark gray, secondary text)
- --text-700: #555555 (medium gray, tertiary text)
- --text-600: #777777 (light gray, disabled/muted text)

DARK MODE NEUTRALS:
- --background-50: #0F0F0F (near black, main background)
- --background-100: #1A1A1A (dark gray, secondary background)
- --background-200: #2D2D2D (medium dark gray, tertiary background)
- --background-300: #404040 (dark gray, borders and dividers)
- --text-900: #FFFFFF (white, primary text)
- --text-800: #E8E8E8 (light gray, secondary text)
- --text-700: #CCCCCC (medium gray, tertiary text)
- --text-600: #999999 (light gray, disabled/muted text)

SEMANTIC COLOR MAPPINGS:
- --color-ai: Glaucous (#5B8DEE / #7BA3F5)
  Purpose: Identifies AI-generated or AI-assisted tasks
  Usage: Task type badges, AI feature indicators, automation markers

- --color-human: Tea Green (#2D5A3D / #7FB069)
  Purpose: Identifies human-created or human-assigned tasks
  Usage: Task type badges, human responsibility indicators

- --color-hitl: Orange Peel (#FF9500 / #FFB84D)
  Purpose: Identifies human-in-the-loop checkpoints
  Usage: Task type badges, approval required indicators, pending actions

- --color-success: Tea Green (#2D5A3D / #7FB069)
  Purpose: Indicates successful completion or positive states
  Usage: Completed task checkmarks, success messages, positive feedback

- --color-warning: Bittersweet (#E74C3C)
  Purpose: Indicates warnings or items needing attention
  Usage: Warning badges, alert messages, urgent items

THEME SWITCHING MECHANISM:
1. User clicks theme toggle in header
2. ThemeProvider updates state and localStorage
3. Document root class changes from "" to "dark"
4. CSS variables automatically switch via media queries and class selectors
5. All components using CSS variables update instantly
6. No component re-renders needed (pure CSS update)

EXAMPLE CSS VARIABLE USAGE:
```css
/* Light mode (default) */
:root {
  --background-50: #FFFFFF;
  --text-900: #1A1A1A;
  --glaucous: #5B8DEE;
}

/* Dark mode */
.dark {
  --background-50: #0F0F0F;
  --text-900: #FFFFFF;
  --glaucous: #7BA3F5;
}

RELATIONAL DATABASE STRUCTURE:

1. USERS TABLE
   Columns:

   - name (VARCHAR)
   - phone (VARCHAR, nullable)
   - location (VARCHAR, nullable)

   - two_factor_enabled (BOOLEAN)
   - data_collection_enabled (BOOLEAN)
   - created_at (TIMESTAMP)
   - updated_at (TIMESTAMP)
   - deleted_at (TIMESTAMP, nullable, soft delete)

2. PROJECTS TABLE
   Columns:
   - id (UUID, primary key)
   - name (VARCHAR)
   - description (TEXT)
   - domain (ENUM: 'Product', 'Marketing', 'Engineering', 'Design')
   - status (ENUM: 'Active', 'Completed', 'On Hold')
   - progress (INTEGER, 0-100)
   - start_date (DATE)
   - end_date (DATE)
   - goals (TEXT)
   - kpis (TEXT)
   - owner_id (UUID, foreign key to users)
   - created_at (TIMESTAMP)
   - updated_at (TIMESTAMP)
   - deleted_at (TIMESTAMP, nullable, soft delete)

3. TASKS TABLE
   Columns:
   - id (UUID, primary key)
   - title (VARCHAR)
   - description (TEXT)
   - type (ENUM: 'AI', 'Human', 'HITL')
   - status (ENUM: 'Pending', 'In Progress', 'Completed')
   - project_id (UUID, foreign key to projects)
   - assignee_id (UUID, foreign key to users, nullable)
   - due_date (DATE, nullable)
   - priority (ENUM: 'Low', 'Medium', 'High')
   - created_at (TIMESTAMP)
   - updated_at (TIMESTAMP)
   - deleted_at (TIMESTAMP, nullable, soft delete)

4. TASK_DEPENDENCIES TABLE
   Columns:
   - id (UUID, primary key)
   - task_id (UUID, foreign key to tasks)
   - depends_on_task_id (UUID, foreign key to tasks)
   - created_at (TIMESTAMP)

5. PROJECT_COLLABORATORS TABLE
   Columns:
   - id (UUID, primary key)
   - project_id (UUID, foreign key to projects)
   - user_id (UUID, foreign key to users)
   - role (ENUM: 'Owner', 'Editor', 'Viewer')
   - joined_at (TIMESTAMP)

6. NOTIFICATIONS TABLE
   Columns:
   - id (UUID, primary key)
   - type (ENUM: 'ProjectUpdate', 'TaskAssignment', 'Completion', 'Comment')
   - title (VARCHAR)
   - message (TEXT)
   - read (BOOLEAN)
   - user_id (UUID, foreign key to users)
   - related_project_id (UUID, foreign key to projects, nullable)
   - related_task_id (UUID, foreign key to tasks, nullable)
   - created_at (TIMESTAMP)
   - deleted_at (TIMESTAMP, nullable, soft delete)

7. ACTIVITIES TABLE
   Columns:
   - id (UUID, primary key)
   - type (ENUM: 'TaskCreated', 'TaskCompleted', 'ProjectCreated', 'CommentAdded')
   - user_id (UUID, foreign key to users)
   - project_id (UUID, foreign key to projects, nullable)
   - task_id (UUID, foreign key to tasks, nullable)
   - description (TEXT)
   - created_at (TIMESTAMP)

INDEXES:
- users.email (unique)
- projects.owner_id
- projects.status
- tasks.project_id
- tasks.assignee_id
- tasks.status
- notifications.user_id
- notifications.read

19. TROUBLESHOOTING GUIDE
================================================================================

COMMON ISSUES & SOLUTIONS:

1. THEME NOT PERSISTING
   Problem: Theme preference not saved after refresh
   Solution:
   - Check localStorage is enabled in browser
   - Clear browser cache and cookies
   - Check ThemeProvider is wrapping entire app
   - Verify CSS variables are defined in globals.css

2. NOTIFICATIONS NOT APPEARING
   Problem: Notifications dropdown is empty
   Solution:
   - Check notification data in mock data
   - Verify NotificationPanel component is imported
   - Check browser console for errors
   - Verify notification bell icon is visible

3. LAYOUT ISSUES
   Problem: Sidebar or header not displaying correctly
   Solution:
   - Check LayoutWrapper is used on page
   - Verify CSS variables are applied
   - Check responsive breakpoints
   - Clear browser cache

4. FORM SUBMISSION ERRORS
   Problem: Form not submitting or showing errors
   Solution:
   - Check form validation logic
   - Verify input fields have proper names
   - Check console for JavaScript errors
   - Verify form handler is attached

5. PERFORMANCE ISSUES
   Problem: App is slow or laggy
   Solution:
   - Check browser DevTools Performance tab
   - Verify no infinite loops in useEffect
   - Check for unnecessary re-renders
   - Optimize images and assets
   - Check network requests

================================================================================
20. GLOSSARY & TERMINOLOGY
================================================================================

TERMS & DEFINITIONS:

1. AI TASK
   A task that is primarily handled by artificial intelligence. These tasks
   are generated by the AI system and may require human review or approval.

2. HUMAN TASK
   A task that requires human effort and decision-making. These tasks are
   assigned to team members and tracked for completion.

3. HITL (Human-In-The-Loop)
   A task that combines AI assistance with human oversight. The AI provides
   suggestions or partial solutions, and humans make final decisions.

4. PROJECT
   A collection of related tasks with a common goal. Projects have a status,
   timeline, and team of collaborators.

5. WORKFLOW
   The sequence of tasks and their dependencies in a project. Visualized as
   a flowchart showing task relationships.

6. NOTIFICATION
   An alert sent to a user about project or task updates. Can be marked as
   read or deleted.

7. ACTIVITY
   A record of an action taken in the system. Activities are logged for
   audit and tracking purposes.

8. COLLABORATOR
   A team member who has access to a project. Can have different roles
   (Owner, Editor, Viewer).

9. THEME
   The visual appearance of the application. CollabFlow supports light and
   dark themes with persistent user preferences.

10. COMPONENT
    A reusable UI element that encapsulates structure, style, and behavior.
    Components are the building blocks of the application.

11. HOOK
    A React function that lets you use state and other React features in
    functional components. Examples: useState, useEffect, useContext.

12. CONTEXT
    A React API for passing data through the component tree without prop
    drilling. Used for theme and global state management.

13. MOCK DATA
    Fake data used for development and testing. Replaces real API calls
    during development.

14. RESPONSIVE DESIGN
    Design that adapts to different screen sizes and devices. CollabFlow
    is fully responsive.

15. ACCESSIBILITY
    The practice of making applications usable by everyone, including people
    with disabilities. CollabFlow meets WCAG 2.1 AA standards.
